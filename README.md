# Project Memory Skill

`Project Memory` 是一个给桌面 Agent 使用的 Skill，用来帮助 Agent 持续整理和维护单个项目的重要信息。

它的重点不是“记录所有过程”，而是把一个项目里真正会在后面继续影响理解和协作的内容整理出来，例如：

- 项目背景
- 当前目标
- 当前进展
- 关键决策
- 重要约束
- 关键文件或链接
- 后续待办
- 下次继续时最该先看的内容

这样做的目的，是让你和 Agent 在隔了一段时间之后，仍然能快速接上同一个项目，而不是每次都重新回忆、重新梳理。

## 这个 Skill 是做什么的

这个 Skill 会教 Agent：

- 判断什么信息值得留下，什么信息不用记
- 把零散聊天、群聊讨论和项目细节整理成一份清晰的项目资料
- 在已有项目资料的基础上持续更新，而不是反复新增重复内容
- 在信息不清楚时，先向你确认，再写进项目资料
- 记录这份项目资料存放在哪里，方便后续继续维护

你可以把它理解成：

“让 Agent 帮你维护一份单项目的长期资料，让这个项目不会因为聊天太长、时间隔太久、信息太零散而断掉。”

## 什么情况下适合使用

如果你有下面这些需求，这个 Skill 就很适合：

- 你刚开始一个新项目，想先把背景、目标和当前状态整理出来
- 你们已经聊了很多，想把有价值的信息沉淀下来
- 你想把群聊里和项目有关的讨论整理成后续还找得到、看得懂的内容
- 你隔了一段时间重新看这个项目，希望快速找回上下文
- 你希望 Agent 在聊天过程中顺手帮你维护项目资料
- 你刚刚确定了一个关键决策、限制条件、文件位置或待办，想补充进去
- 你希望下次继续时，一打开就知道应该先看什么、从哪里接上

## 这个 Skill 不适合什么

这个 Skill 不适合用来：

- 管理多个项目的总览
- 记录完整会议纪要
- 保存所有聊天原文
- 做细颗粒度的任务排期系统
- 替代正式的项目管理平台

它更适合做一件事：

为一个项目保留一份简短、可靠、持续可更新的资料。

## 它会产出什么

通常会产出一份单项目资料，里面包括：

- 这是什么项目
- 为什么要做
- 现在目标是什么
- 当前进行到哪里
- 哪些决策已经确定
- 有哪些约束条件
- 哪些文件或链接最重要
- 接下来要做什么
- 下次继续时应该先看什么

## 你可以怎么理解它的价值

对人来说，这个 Skill 的价值不是“多一个文档”。

它真正解决的是这些问题：

- 聊天太多，重要信息被冲散
- 过了几天之后，不知道之前定了什么
- 群里讨论过，但找不到结论
- Agent 每次都要重新理解项目
- 项目没有正式文档，但又确实需要一个稳定的整理入口

如果你遇到的是这些问题，这个 Skill 基本就是为这种场景准备的。
